  Manual Setup in GCP Console:
   1. Enable APIs: Ensure Cloud Build, Cloud Run, and Container Registry APIs are enabled in your GCP project.
   2. Service Account Permissions: The Cloud Build service account (usually
      [PROJECT_NUMBER]@cloudbuild.gserviceaccount.com) needs the Cloud Run Admin and Service Account User roles to
      perform the deployment.
   3. Firestore: Ensure your Firestore database is initialized in Native Mode.
   4. Firebase Web Config: Ensure you have added a Web App to your Firebase project and enabled Email/Password, Google,
      and Facebook authentication.


  Your application will be live at the URL provided by Cloud Run after the build completes successfully!

  1. Access the GCP Console
   1. Go to the Google Cloud Console (https://console.cloud.google.com/).
   2. Log in with the same Google account you use for Firebase.
   3. In the project selector (top-left dropdown), select the project that matches your Firebase Project Name/ID.


  2. Enable Required APIs
  For this specific deployment (Docker + Cloud Run), you must enable these services in GCP:
   1. Search for "APIs & Services" in the top search bar.
   2. Click "+ ENABLE APIS AND SERVICES".
   3. Search for and enable each of these:
       * Cloud Run API: To host your containerized Flutter app.
       * Cloud Build API: To run the cloudbuild.yaml pipeline.
       * Artifact Registry API: To store your Docker images (this is the modern version of Container Registry).


  3. Configure IAM Permissions (Critical)
  Cloud Build needs permission to deploy to Cloud Run on your behalf:
   1. Go to IAM & Admin > IAM.
   2. Find the service account ending in @cloudbuild.gserviceaccount.com.
   3. Click the Edit (pencil) icon.
   4. Click "+ ADD ANOTHER ROLE" and add these two:
       * Cloud Run Admin
       * Service Account User
   5. Click Save.


  4. Set Up Authentication in Firebase
  Even though you are deploying via GCP, you must configure your sign-in methods in the Firebase Console
  (https://console.firebase.google.com/):
   1. Go to Build > Authentication > Sign-in method.
   2. Enable Email/Password, Google, and Facebook.
   3. For Google SSO: Add your Cloud Run URL (once deployed) to the "Authorized domains" list in Firebase Auth settings
      so the login redirect works.


  5. (Optional) Artifact Registry Setup
  If you want to use the newer Artifact Registry (recommended over Container Registry):
   1. Go to Artifact Registry in GCP.
   2. Create a repository named cloud-run-source-manual (or similar).
   3. Type: Docker.
   4. Region: us-central1 (to match your cloudbuild.yaml).


  Once these steps are done, your GCP environment is fully prepared to receive the gcloud builds submit command I
  provided earlier!
